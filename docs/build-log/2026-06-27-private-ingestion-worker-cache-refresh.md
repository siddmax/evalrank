# Private Ingestion-Worker Cache Refresh

Date: 2026-06-27

Scope: private Syndai EvalRank worktree only. This public log records the public-safe status update; no private schema data, private fixtures, customer data, hosted runtime code, secrets, or Syndai implementation files were copied into the public EvalRank repo.

## What Changed Privately

- The private EvalRank active coding-router script chain is now available as a replayable worker refresh helper.
- The weekly private eval pipeline worker calls that refresh after live non-dry measurement and optional promotion.
- Dry-run worker cycles skip the expensive refresh path.
- The refresh derives deterministic active, candidate, and pooled calibration snapshot ids.
- The refresh rebuilds active item responses, contamination checks, candidate-ledger calibration rows, pooled IRT calibration, Stage-2 rows with explicit pooled calibration, Stage-4 rows, and the materialized cache.
- The refresh fails closed if pooled IRT does not produce publishable calibration rows, preserving the previous cache instead of publishing uncalibrated optimistic rankings.

## Product Decision

Long-term EvalRank should serve cached recommendations generated by private batch workers. Request-path live evals, IRT fitting, and model adjudication are the wrong UX and cost shape: they add latency, spend, and nondeterminism at the moment the user needs a fast, explainable recommendation.

The chosen architecture pays expensive work offline, keeps cached reads fast, and makes calibration comparability explicit. Accuracy and UX are prioritized over small batch-compute cost; request latency and performance benefit because the hosted surface can later read the verified cache.

## Public Boundary

The public EvalRank repo still owns product-neutral contracts, schemas, SDKs, CLI, MCP boundary, examples, and method notes. Private Supabase/Postgres migrations, source adapters, worker orchestration, calibration rows, scorer rows, and cache rows remain in Syndai's private `evalrank` schema until EvalRank has its own deploy/release path or dedicated Supabase project.
